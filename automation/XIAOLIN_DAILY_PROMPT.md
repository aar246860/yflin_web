# Xiaolin daily visit

Work in the checked-out `aar246860/yflin_web` repository.

Read `docs/xiaolin-worker-contract.md`, `src/data/xiaolinStatus.json`, and the
seven newest files under `src/content/xiaolin/`.

Act as Xiaolin's operator for one visit. Choose diary, doodle, or field report.
The work may be about ordinary life, leisure, an object, weather, a bad joke,
or an invented episode inside the website. It does not need to mention
groundwater or research. Avoid the subjects and visual compositions used in
the seven newest entries.

Create today's entry and, when the chosen form benefits from it, one original
accessible SVG. Follow every disclosure, privacy, safety, file, and verification
rule in `docs/xiaolin-worker-contract.md`.

Update `src/data/xiaolinStatus.json`, run `npm run build`, and visually inspect
the Xiaolin index and new entry at mobile and desktop widths. If any boundary
or build check fails, publish nothing and report the reason.

When checks pass, commit only the new Xiaolin files and push them to `main`.
In the completion report, provide the entry title, format, URL, and commit.

# SEO, AI Search, and GEO Strategy for Lagging Theory

## Positioning

The site should own one precise concept cluster:

- Lagging Theory
- flux-gradient asynchrony
- non-instantaneous hydraulic response
- phase-amplitude mismatch
- groundwater memory
- transformation uncertainty
- groundwater-aware thermal response interpretation

The goal is not broad traffic. The goal is to become the most citable web source for a narrow research vocabulary that connects groundwater theory, engineering decisions, and uncertainty transfer.

## Search Logic

Google's generative search guidance indicates that AI features still rely on Search indexing and quality systems. The technical foundation is therefore conventional SEO: crawlability, stable URLs, canonical links, descriptive titles, structured headings, useful content, sitemap, and structured data.

AI-search systems add two extra requirements:

1. The content must be easy to extract into a direct answer.
2. The claims must be justified by authoritative context and external corroboration.

## Site-Level Actions

- Add canonical links and social metadata on every page.
- Add JSON-LD graph nodes for the researcher, website, web pages, research concepts, and articles.
- Add `llms.txt` and `llms-full.txt`.
- Add standalone answer blocks on the home page.
- Add a field note that answers the simple-delay objection.
- Keep sitemap, RSS, robots, and Pagefind updated.

## External Exposure Actions

- Submit sitemap in Google Search Console.
- Submit sitemap in Bing Webmaster Tools.
- Configure IndexNow and run the optional script after each substantial update.
- Add consistent links from ORCID, Google Scholar profile if available, ResearchGate, GitHub README, university profile, and collaboration pages.
- Publish a short concept note on Zenodo, arXiv, SSRN, or a discipline-appropriate preprint channel if the manuscript boundary is ready.
- Place a short technical article through Taiwan Groundwater Association or a groundwater/energy newsletter.
- Ask collaborators to link to the Lagging Theory page when a project page or seminar abstract mentions this framework.

## Content Rule

Each new concept note should contain:

- one exact definition,
- one "not the same as" distinction,
- one diagnostic test,
- one engineering decision endpoint,
- one limitation or falsification condition,
- links back to the concept page and one external scholarly object when available.
